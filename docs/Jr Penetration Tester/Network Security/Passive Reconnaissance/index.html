#Network

Three command-line tools

  • whois to query WHOIS servers
  • nslookup to query DNS servers
  • dig to query DNS servers These are all publicly available records and hence do not alert the target.

Two online services

  • DNSDumpster
  • Shodan.io These two online services allow us to collect information about our target without directly connecting to it.

Passive Versus Active Recon

Reconnaissance (recon) can be defined as a preliminary survey to gather information about a target.

We divide reconnaissance into Passive Reconnaissance and Active Reconnaissance.

Whois

WHOIS is a request and response protocol that follows the RFC 3912 specification. A WHOIS server listens on TCP port 43 for incoming requests.

We can learn four parts

  • Registrar
  • Contact info of registrant
  • Creation, update, and expiration dates
  • Name Server

By using whois DOMAIN_NAME to get information.

nslookup and dig

We can find the IP address of a domain name using nslookup , which stands for Name Server Look Up.

The syntax is nslookup OPTIONS DOMAIN_NAME SERVER

  • OPTIONS: it contains the query types, which is shown below
  • DOMAIN_NAME
  • SERVER: it is the DNS server we want to query

Cloudflare offers 1.1.1.1 and 1.0.0.1, Google offers 8.8.8.8 and 8.8.4.4, and Quad9 offers 9.9.9.9 and 149.112.112.112. more public DNS server

Query typeResult
AIPv4 Addresses
AAAAIPv6 Addresses
CNAMECanonical Name
MXMail Servers
SOAStart of Authority
TXTTXT Records

To get the IPv4 address, we can use nslookup -type=A tryhackme.com 1.1.1.1

To get the IPv6 address, we can use nslookup -type=AAAA tryhackme.com 1.1.1.1

To learn about the email servers and configurations for a particular domain, we can use nslookup -type=MX tryhackme.com

Example:

user@TryHackMe$ nslookup -type=MX tryhackme.com
Server:		127.0.0.53 
Address:	127.0.0.53#53

Non-authoritative answer: 
tryhackme.com	mail exchanger = 5 alt1.aspmx.l.google.com. tryhackme.com	mail exchanger = 1 aspmx.l.google.com. 
tryhackme.com	mail exchanger = 10 alt4.aspmx.l.google.com. tryhackme.com	mail exchanger = 10 alt3.aspmx.l.google.com. tryhackme.com	mail exchanger = 5 alt2.aspmx.l.google.com.

We can see the current email configuration uses Google. Since MX is looking up the Mail Exchange servers, we notice that when a mail server tries to deliver email @tryhackme.com, it will try to connect to the aspmx.l.google.com, which has order 1. If it is busy or unavailable, the mail server will attempt to connect to the next in order mail exchange servers, alt1.aspmx.l.google.com or alt2.aspmx.l.google.com.

For more advanced DNS queries and additional functionality, we can use dig , the acronym for "Domain Information Groper"

The syntax is dig @SERVER DOMAIN_NAME TYPE dig @1.1.1.1 tryhackme.com MX

Compared to nslookup, dig returned more information, such as the TTL (Time To Live).

DNSDumpster

DNSDumpster

By relying on brute-force queries, it provides detailed DNS queries, returns in tables and a graph, it will provides the subdomain and information about listening server.

Shodan.io

Shodan.io

It can learn various pieces of information about the client’s network, without actively connecting to it.

  • IP address
  • hosting company
  • geographic location
  • server type and version
